# Подготовка Cursor-адаптеров из канона (выполняет агент Cursor)

Инструкция для **агента Cursor**. Запускает её **человек вручную** в Cursor — обычно после `git pull`
или после изменения правил. Цель: из общего канона `.cursor/skills/*` подготовить **Cursor-сторону** —
правила-адаптеры и кастомных агентов — чтобы можно было сразу работать. (Claude-сторона готовится
отдельно скриптом `scripts/project-rules-to-claude.mjs` — её НЕ трогать.)

## Жёсткие правила

1. **Канон не менять.** Файлы `.cursor/skills/*/SKILL.md` — источник истины, только читать.
2. **Создавать или обновлять, не дублировать.** Если адаптер/агент уже есть — обновить на месте.
3. **Брать только нужное:** правило идёт в работу, если в его frontmatter `scope` содержит `cursor`
   и `status: active`. Остальные пропускать.
4. Адаптеры — **тонкие**: триггер + ссылка на канон. Не копировать тело скилла.

## Шаг 1. Правила-адаптеры → `.cursor/rules/<id>.mdc`

Для каждого скилла, у которого `class` = `gate` | `guidance` | `process-invariant` (то есть НЕ
`kind: subagent-spec`), создай/обнови файл `.cursor/rules/<id>.mdc` строго по шаблону:

```
---
description: <возьми frontmatter `applies-when` (или `title`, если короче)>
globs: <массив из frontmatter `globs`; если поля нет — удали эту строку>
alwaysApply: <true, если `globs` отсутствует ИЛИ покрывает весь репозиторий (например ["**/*"]); иначе false>
---

# <title> (адаптер Cursor)

Тонкий адаптер. Источник истины — `.cursor/skills/<id>/SKILL.md` (открыть skill `<id>`).

Короче: <1–2 строки сути из канона своими словами>.
<если class = gate: добавь строку про проверку `scripts/check-gates/<id>.mjs` и текущий режим из `enforcement-level`>.
```

Маппинг полей: `id` → имя файла; `title` → заголовок; `applies-when` → `description`; `globs` → `globs`.

## Шаг 2. Субагенты → кастомные агенты Cursor

Для каждого скилла с `kind: subagent-spec` (сейчас это `unit-test-writer` и `autotest-writer`) создай/обнови
**кастомного агента Cursor** в репозитории: **`.cursor/agents/<id>.md`** — зеркало `.claude/agents/<id>.md`
(тот же frontmatter `name`/`description`, тело со ссылкой на канон `.cursor/skills/<id>/SKILL.md`).

Что должно быть в каждом агенте:
- **Имя:** `<id>` (например `unit-test-writer`).
- **Когда вызывать / описание:** из frontmatter `applies-when`.
- **Инструкция агента:** «прочитай канон `.cursor/skills/<id>/SKILL.md` и следуй ему», плюс ключевые
  правила из тела канона кратко: **независимость** (тесты пишет НЕ автор кода), **дисциплина пирамиды**,
  конвенции стека (для unit — vitest+RTL; для autotest — Playwright, только критичные сквозные пути).
- **Инструменты:** чтение файлов, поиск по коду, запись тест-файлов, запуск тестов.

Не встраивай в агента весь текст канона — ссылайся на него; держи определение компактным.

## Шаг 3. Отчёт

После выполнения коротко перечисли: какие `.mdc` создал/обновил, каких Cursor-агентов создал/обновил,
что пропустил (и почему — например `scope` без `cursor`).
