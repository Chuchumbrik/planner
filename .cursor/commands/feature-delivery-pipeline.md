**Права GitHub:** пользователь предоставил права на `gh`/MCP/issue/PR/push (когда сценарий разрешает) — используй их, пока среда не вернула ошибку. Подробнее — [reference.md](../skills/feature-delivery-workflow/reference.md).

Выполни скилл **`feature-delivery-workflow`** в **режиме субагентов** (см. `.cursor/skills/feature-delivery-workflow/SKILL.md`, **«Права GitHub и git для агента»**, шаблоны в `.cursor/skills/feature-delivery-workflow/pipeline-agents.md`).

**Контекст фичи (заполни или вставь ТЗ):**

- Название и цель (что пользователь получит).
- Что **не** входит в scope этой итерации.
- Источник: issue / чат / документ.
- Ограничения: срок, не пушить, только план и т.п.

**Оркестрация:** выбери субагентов из `.cursor/agents/feature-*.md` по `pipeline-agents.md`. **Сначала** блок **`## [ВОПРОСЫ] По фиче`** (обязателен: либо «не требуются» с обоснованием, либо список вопросов с вариантами).

В конце любого сценария — **итоговое сообщение** по `pipeline-agents.md`: суть фичи + таблица (включая **Уточняющие вопросы**, **Ручная настройка**); при необходимости — блок ручной настройки с `- [ ]`.

---

### Краткие вызовы

**Только план и вопросы** — без коммитов кода (можно доки по согласованию):

`/feature-delivery-pipeline` + контекст фичи. Режим: вопросы → PRODUCT+SYSTEM → `[FEATURE_DECISION]`; стоп до реализации без явного «делаем код».

**Полный цикл с push** — после `[FEATURE_DECISION]` = `IMPLEMENT_NOW`:

`/feature-delivery-pipeline` + контекст. Режим: полный цикл; перед веткой `git fetch`, `checkout main`, `pull`; ветка `feature/<slug>-…`, доки до кода, UX→FE→BE→QA, PR в `main`, в теле PR при наличии issue — `Fixes #N`.

**Только уточняющие вопросы** — если контекст сырой:

`/feature-delivery-pipeline` + черновик идеи. Режим: только блок `[ВОПРОСЫ]` и краткое резюме, что ждём от пользователя.
