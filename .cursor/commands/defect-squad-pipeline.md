**Права GitHub:** пользователь предоставил права на `gh`/MCP/issue/PR/push (когда сценарий разрешает) — используй их, пока среда не вернула ошибку. Подробнее — [reference.md](../skills/github-defect-workflow/reference.md).

Выполни скилл **`github-defect-workflow`** в **режиме субагентов** (см. `.cursor/skills/github-defect-workflow/SKILL.md`, в т.ч. **«Права GitHub и git для агента»**, шаблоны в `.cursor/skills/github-defect-workflow/pipeline-agents.md`).

**Контекст дефекта (заполни или приложи issue):**

- Issue: #… / URL
- Суть и шаги воспроизведения (или почему информативный):
- Ограничения — см. **краткие варианты** ниже.

**Оркестрация:** сам выбери субагентов из `.cursor/agents/` и порядок по **`.cursor/skills/github-defect-workflow/SKILL.md`** и **`pipeline-agents.md`**; не проси пользователя перечислять роли.

В конце любого сценария — **итоговое сообщение** по `pipeline-agents.md`: блок **«Суть дефекта (кратко)»** + таблица (включая **Ручная настройка**); при необходимости — отдельный блок **«Ручная настройка после выката»** с чекбоксами `- [ ]` для человека, как в шаблоне `pipeline-agents.md`.

### Краткие вызовы (подставь `#N`)

**Только разбор** — до `[DECISION]`, без правок файлов и без коммита:

`/defect-squad-pipeline` Issue **#N** + вставь тело/шаги issue. Режим: только анализ; стоп после `[DECISION]`.

**Сухой прогон** — полный пайплайн локально, без push и без закрытия issue без моего «ок»:

`/defect-squad-pipeline` Issue **#N** + вставь тело issue. Режим: можно править и коммитить локально, **не** пушить; issue **не** закрывать без моего явного подтверждения.

**Боевой фикс** — до `[QA]` PASS, затем ветка от `main`, push, **PR в `main`** с **`Fixes #N`** в описании (автозакрытие issue), фаза D по скиллу:

`/defect-squad-pipeline` Issue **#N** + вставь тело issue. Режим: полный цикл, **перед веткой** — `git fetch`, `checkout main`, `pull`/`merge origin/main`, при необходимости stash чужого WIP; ветка `fix/issue-<N>-…` от актуального `main`, push, PR в `main` с первой строкой описания `Fixes #N`, фаза D и комментарий в issue со ссылкой на PR.
