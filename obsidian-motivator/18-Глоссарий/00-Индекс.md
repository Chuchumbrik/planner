# Глоссарий — переменные и методы

> Поддерживаемые определения полей и алгоритмов продукта «Мотиватор». В документации `obsidian-motivator/` каждое упоминание переменной или метода оформляется wikilink с якорем **`#Определение`** — при наведении Obsidian показывает **определение** из callout на странице глоссария (нужен включённый core-плагин **Page preview**).

**Версия документа:** 1.6  
**Дата:** 2026-05-30

---

## Подсказка при наведении (Obsidian)

| Элемент | Значение |
|---------|----------|
| Формат ссылки | `[[scheduledLocalDate#Определение\|scheduledLocalDate]]` (в **таблицах** символ `\|` перед псевдонимом обязателен) |
| Якорь на странице | `^glossary-def` стоит **на отдельной строке после** callout «Определение» (пустая строка между callout и якорем). **Не** помещать `^glossary-def` внутрь строки callout — Obsidian **не** показывает превью для ссылок внутрь callout |
| Поля объекта / результата | `[[doubleConfirmPending#^glossary-def-localDate\|localDate]]` — блок после callout на странице родителя |
| Корень vault | **`planner`** (репозиторий) или **`obsidian-motivator`** — см. [[00-Obsidian-vault]]; ссылки **без** префикса `18-Глоссарий/…` |
| Превью | **Page preview** в `.obsidian` корня vault (`planner/.obsidian` при vault = репозиторий) |
| Режим редактирования | По умолчанию **Ctrl/Cmd** + наведение; без модификатора — **Настройки → Page preview** (см. [[00-Obsidian-vault]]) |
| Режим чтения | Обычное наведение |
| Скрипты | `node obsidian-motivator/18-Глоссарий/_scripts/fix-block-ids.mjs`, `…/migrate-universal-links.mjs` |

## Как добавить запись

1. Скопировать [[переменная]] или [[метод]] в `переменные/` или `методы/` (имя файла = идентификатор).
2. Заполнить **frontmatter `description`**, callout **Определение** (текст **должен совпадать** с `description`) и блок `^glossary-def`.
3. Заполнить типы данных и разделы «Применение — общее / техническое».
4. Добавить строку в таблицу выше (ссылка **с `#Определение`**).
5. В документах-источниках: `[[идентификатор#Определение|идентификатор]]` (короткое имя файла, уникально во vault `planner`).
