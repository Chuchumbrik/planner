## 1. Стратегический выбор

**MVP = Вариант А:** полноценное планирование по датам + визуальный календарь + полноценные отчёты (на клиенте, см. [[12-Журнал-решений#DR-007 — Отчёты где считать аналитику]]).

---

## 2. Обязательный Scope MVP

### Основные сущности

- **Задача**
- **Группа / цвет**
- **Приоритет** — настраиваемый **1–5** (+ пользовательские названия уровней)
- **План работы над задачей** — чек-лист (продуктовый смысл ранее «подзадачи»)
- **Backlog**

### Планирование и просмотр

| Период | Представление |
|--------|----------------|
| **День** | Список + мини-карточки |
| **Неделя** | Таймблокинг: горизонталь — дни, вертикаль — часы; задачи в реальных временных слотах |
| **Месяц** | Календарь с отметками + фильтры отображения (группы, приоритеты, повторы) |

### Ключевые механики

- Поле **«Оценка времени»** (необязательное), в минутах.
- **Таймблокинг:** при указании времени начала + оценки задача занимает слот `[начало, начало + оценка]`; при завершении задачи — освобождение слота (обратная логика по продуктовым правилам).
- **Пересечения:** проверка при планировании + **мягкое предупреждение** (пример: «Задача пересекается с другой на 25 минут. Продолжить?»).
- **Повторы (простые):** каждый день; каждые N дней; по дням недели.
- **Двойное подтверждение**, **End-of-Day**, **анимации**, **клиентское шифрование** — в объёме MVP.

---

## 3. Продуктовые правила MVP

- **День** — локальная календарная дата устройства.
- **Backlog** — отдельная секция в Today.
- **Стрик** — см. согласованное правило в [[12-Журнал-решений#DR-013 — Стрик: уточнение под MVP v1.0 и DR-001]] (EOD + задачи плана дня).
- **Время задачи** — две **взаимоисключающие** опции: «время начала» **или** «время завершения».
- **Приоритеты** — полностью настраиваемые (**1–5 + названия**).
- **Фильтры** — только для текущего просмотра + **информер** (сводка по активным фильтрам / охвату).
- **Удаление аккаунта** — с возможностью восстановления в течение **30 дней** по ключу (детали реализации — серверная политика + UX, см. план реализации).

---

## 4. Обязательные улучшения в MVP (UX / структура)

- Разделить **просмотр** (мини-карточка) и **редактирование**.
- Выбор цвета — **палитра**.
- Фильтры + информер.
- Настройки — **по группам**.
- Кнопка удаления аккаунта.
- Настраиваемая система приоритетов (1–5).
- Подзадачи по смыслу MVP → **«План работы над задачей»** (чек-лист).

---

## 5. Что НЕ входит в MVP (v1.1+)

- Глобальные цели с метрикой прогресса.
- Шаблоны задач + конструктор полей.
- Информационные справки по долгому нажатию.
- Геймификация, AI и т.п.
