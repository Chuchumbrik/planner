# 21 — Phase 13: Дизайн-layout (3-зонная структура) — 🚧 В процессе

**Статус:** согласован мок 2026-06-22 (до кода). Источник решений — [[19-Business-требования]] BR-D-007(D), BR-D-008, BR-D-009, BR-D-010, BR-D-011, BR-D-012. Поправка по аудитории — [[02-Целевая-аудитория]] (персона «пользователь с регулярным расписанием»; СДВГ — одна из персон, не вся ЦА).

**Референс:** Dribbble «Healthcare Dashboard: Calendar / Staff View» (Gotoinc) — берём РАСПОЛОЖЕНИЕ БЛОКОВ и идею цветных карточек по группам. Цвета — свои (Motivator 2.0, primary изумруд `#4edea3`), палитру референса (индиго) НЕ копируем.

## Как работает (целевая структура)

3-зонный layout, заполняющий ширину экрana: **икон-рейл (~56px)** + **левая панель (~280px)** + **главная зона (flex)** + **правый сайдбар по триггеру (~360px overlay)**.

| Блок | Содержимое | Закрывает (аудит) |
|------|-----------|-------------------|
| Икон-рейл 56px | навигация (Планировщик/Отчёты/превью/Админ/Настройки), account снизу; метки в tooltip | заменяет текущий 264px-сайдбар |
| Левая панель 280px | мини-календарь (навигатор), **Categories** — цветные фильтр-тоглы по группам (BR-D-011), **Сегодня** — агенда списком | фильтры-модалка → постоянная панель; пустые поля >1200px |
| Главная зона | тулбар (‹Эта неделя›, диапазон, сегмент Д/Н/М/Г); шапка дней крупными чипами; **мягкая** сетка (BR-D-007 D) | пустая 24ч-сетка, обрезанная шапка повторов |
| Правый сайдбар | создание/правка задачи по триггеру (клик на задачу/слот/«+»); вкладки Задача/Событие; Время·Группа·Приоритет·Заметка (BR-D-012) | заменяет модалку; overlay, не сжимает главную |

### Неделя — «мягкая» сетка (BR-D-007, вариант D)
- Ночные часы (0:00–7:00, 22:00–24:00) **схлопнуты** со счётчиком «+N событий» и раскрытием в клик. **Счётчик overflow — обязателен в MVP** (иначе схлоп скрывает события — жёсткое условие devil-advocate).
- Задачи без времени и повторы — **отдельная дорожка НАД сеткой** (не шапка, не подвал).
- Задачи в слотах — **цветные карточки по группам** (`TASK_COLOR_HEX`); пустой слот — инлайн «+ Add» (тайм-блокинг).
- Полюса персон: у кого расписание есть — польза; у кого нет — не давит.

### Месяц (BR-D-008) — цветные точки групп
Точки = те же цвета, что фильтры слева и карточки Недели → единая система чтения. Прошлый долг приглушается (BR-D-009), не «стена красного».

## Согласованные параметры (2026-06-22)

| Параметр | Значение |
|---|---|
| Breakpoint коллапса левой панели | **1280px** (ниже — drawer) |
| «Ночные» часы для схлопа | **0:00–7:00 и 22:00–24:00** (настраиваемо — позже) |
| Икон-рейл vs 264px-сайдбар | **рейл 56px**, метки в tooltip/раскрытии |
| Неделя на мобилке | **горизонтальный свайп** по дням (1 день), не сжатие 7 колонок |
| Правый сайдбар по умолчанию | **закрыт** (триггерится), иначе на 1280px главной зоне остаётся ~584px |

## Деградация по ширине
- **≥1280px:** все 3 зоны; правый сайдбар — overlay по триггеру.
- **768–1024px:** левая панель → drawer по кнопке ☰; сайдбар → overlay на всю высоту.
- **<768px:** bottom-nav (как сейчас); левая панель → полноэкранный drawer; создание/правка → **bottom sheet**; Неделя — свайп по дням.

## План реализации (финализирован 2026-06-23)

Bundle на ветке `feat/phase13-layout`, мердж одним PR после прогона тестов (жёсткое условие дебатов: частичная выкатка = «ломаный» промежуточный state). Единый источник цвета — `TASK_COLOR_HEX` (`packages/motivator-core/src/vault/colors.ts`).

### Принятые решения (две развилки)
- **Цвет группы → миграция vault.** `TaskGroup` (`packages/motivator-core/src/vault/types.ts:2` = `{id,name,sortOrder}`) получает `colorKey: TaskColorKey`. Группа = цвет; задача **наследует** цвет группы, с возможностью оверрайда (`task.colorKey` остаётся). Миграция: существующим группам проставить `colorKey` (по умолчанию по `sortOrder` из палитры или `zinc`); добавить выбор цвета в форму создания/правки группы. Так «цвет тогла Categories = карточки Недели = точки Месяца» становится единой системой (BR-D-011/008).
- **Мобильная Неделя — в этот bundle.** Отдельный однодневный свайп-вид `WeekDayView` на `<768px` (свайп по дням, не сжатие 7 колонок). Фикс «7 колонок в 40px».

Каждый этап — отдельный коммит; откат = revert PR целиком. Модель — аддитивно (`taskColors` опционально; `TaskGroup.colorKey` с дефолтом). Старые `CreateTaskModal`/`TaskEditModal` помечаем deprecated, физически не удаляем до приёмки.
